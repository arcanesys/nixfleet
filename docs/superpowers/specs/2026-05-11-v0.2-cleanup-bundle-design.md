# v0.2 Cleanup Bundle — Design Spec

**Date:** 2026-05-11
**Repos affected:** `nixfleet` (primary), `nixfleet-demo` (consumer), `fleet` (comment touchups)
**Branch:** `feat/v0.2-cleanup-bundle`

## Goal

Ship v0.2 with a tighter operator surface and cleaner crate internals
before v0.3 RFCs (0004-0007) add ~3-6k LOC on top. Four structural
cleanups bundled in one nixfleet PR; two trivial coordinated PRs in
`nixfleet-demo` and `fleet` adopt the renames.

Net effect on default build: byte-identical except for operator binary
names. Net effect on `${pkgs.nixfleet-cli}/bin/`: three fewer binaries
(`nixfleet-mint-token`, `nixfleet-mint-operator-cert`,
`nixfleet-derive-pubkey`), one fewer in `src/` accounting (test file
moved to `tests/`), one optional dependency family behind a feature
flag.

## Non-goals

- **Folding `nixfleet-canonicalize` or `nixfleet-verify-artifact` into
  the umbrella.** Both are the offline-auditor surface; deps stay lean
  (no `reqwest` / `tokio`) so a third-party regulator can verify
  artifacts without pulling the operator network stack. Property
  preserved deliberately.
- **Folding `nixfleet-release` into the umbrella.** Fleet CI invokes
  it by binary name from a workflow that won't move; CI-only tool;
  no operator-UX benefit to folding.
- **Folding `nixfleet-agent` or `nixfleet-control-plane`.** Daemons.
- **Consolidating `observed_view.rs` + `observed_projection.rs`.**
  Distinct intentional code paths: DB-backed (HTTP routes + dispatch)
  vs in-memory projection (reconciler tick). Both consumed by the
  reconciler within a single tick. Merging would lose load-bearing
  separation.
- **Auditing `dispatch.rs` vs `server/checkin_pipeline/dispatch_target.rs`.**
  Investigation showed no overlap — pure-function `decide_target` vs
  per-checkin orchestration. Names overlap; jobs don't.
- **Transport abstraction for RFC-0007 bundle import.** Polling tree
  already factored cleanly (`signed_fetch` primitive +
  `SignedArtifactPoller` skeleton + per-artifact poll modules). Premature
  to abstract before the second transport ships.

## Scope

Four items, all in one nixfleet PR.

### F1 — Operator binary fold

**Today (4 binaries in `nixfleet-cli`):**

```
nixfleet                       (status, rollout trace, config init)
nixfleet-mint-token            (operator-side bootstrap-token mint)
nixfleet-mint-operator-cert    (operator-side mTLS client cert mint)
nixfleet-derive-pubkey         (orgRootKey scratch helper)
```

**After (1 binary):**

```
nixfleet
  status            (existing)
  rollout trace     (existing)
  config init       (existing)
  mint-token        (folded from nixfleet-mint-token)
  mint-operator-cert (folded from nixfleet-mint-operator-cert)
  derive-pubkey     (folded from nixfleet-derive-pubkey)
```

**Cargo.toml changes** (`crates/nixfleet-cli/Cargo.toml`):

```toml
# Remove these three [[bin]] entries:
[[bin]]
name = "nixfleet-mint-token"
path = "src/bin/mint_token.rs"

[[bin]]
name = "nixfleet-mint-operator-cert"
path = "src/bin/mint_operator_cert.rs"

[[bin]]
name = "nixfleet-derive-pubkey"
path = "src/bin/derive_pubkey.rs"
```

**Source structure:**

- `src/bin/mint_token.rs` (334 LOC) → `src/commands/mint_token.rs`
  exposing `pub fn run(args: MintTokenArgs) -> Result<()>`.
- `src/bin/mint_operator_cert.rs` (112 LOC) → `src/commands/mint_operator_cert.rs`
  exposing `pub fn run(args: MintOperatorCertArgs) -> Result<()>`.
- `src/bin/derive_pubkey.rs` (27 LOC) → `src/commands/derive_pubkey.rs`
  exposing `pub fn run(args: DerivePubkeyArgs) -> Result<()>`.
- `src/bin/nixfleet.rs` (158 LOC) extends its `clap` enum with three
  new subcommand variants and dispatches to the `run` functions.

The existing `operator_cert::mint_operator_cert(...)` library function
in `crates/nixfleet-cli/src/operator_cert.rs` is unchanged; only the
binary entry point moves into the umbrella.

**Subcommand surface (clap):**

```rust
#[derive(clap::Subcommand)]
enum Cmd {
    Status(StatusArgs),
    Rollout(RolloutArgs),
    Config(ConfigArgs),
    MintToken(commands::mint_token::Args),
    MintOperatorCert(commands::mint_operator_cert::Args),
    DerivePubkey(commands::derive_pubkey::Args),
}
```

Subcommand names use kebab-case to match Unix convention and avoid
mid-stream rename for ergonomics.

**LOC absorbed:** 27 + 112 + 334 = 473 LOC of bin entry points
collapse into the umbrella; no logic changes.

### F2 — Move `lifecycle_parity_tests.rs` out of `src/`

**Today:** `crates/nixfleet-control-plane/src/lifecycle_parity_tests.rs`
(347 LOC, `#![cfg(test)]` gated). Sits at the module root; visible in
`src/` tree but unreachable from any non-test code path. Forces every
reader of the crate's source layout to context-switch.

**After:** `crates/nixfleet-control-plane/tests/lifecycle_parity.rs`.
Standard Rust integration-test location. Imports `nixfleet_control_plane::*`
public items rather than `crate::*`.

**Mechanics:**

```bash
# F2-1
git mv crates/nixfleet-control-plane/src/lifecycle_parity_tests.rs \
       crates/nixfleet-control-plane/tests/lifecycle_parity.rs

# F2-2: drop the `mod lifecycle_parity_tests;` declaration in lib.rs
# F2-3: change `use crate::*` → `use nixfleet_control_plane::*`
#       inside the moved file. Only public items are reachable from
#       integration tests; if anything used was crate-private, surface
#       it via `pub(crate)` → `pub` deliberately, or move that helper
#       into a `pub mod test_helpers` (mirroring `db/test_helpers.rs`).
```

**Risk:** Low. The file is already test-only. The risk is that
`crate::*` reaches a `pub(crate)` helper that needs surfacing.
Validate by `cargo test -p nixfleet-control-plane --test lifecycle_parity`.

### F3 — Feature-gate Prometheus metrics

**Today:** `metrics` + `metrics-exporter-prometheus` are mandatory deps
of `nixfleet-control-plane`. The lab Prometheus job scrapes
`/metrics`; everyone else pays the dep weight.

**After:** `metrics` feature in `Cargo.toml`, defaulted on so lab CI
stays compatible:

```toml
[features]
default = ["metrics"]
metrics = ["dep:metrics", "dep:metrics-exporter-prometheus"]

[dependencies]
metrics = { version = "0.24", optional = true }
metrics-exporter-prometheus = { version = "0.18", default-features = false, optional = true }
```

**Code-side `cfg` annotations** (4 sites):

- `crates/nixfleet-control-plane/src/metrics.rs` — wrap module body in
  `#[cfg(feature = "metrics")]` and provide no-op stubs for the
  `record_*` helpers when the feature is off, so call sites stay
  unchanged.
- `crates/nixfleet-control-plane/src/server/routes/metrics.rs` —
  `#[cfg(feature = "metrics")]` on the handler; alternate
  no-features handler returns 404.
- `crates/nixfleet-control-plane/src/server/mod.rs:63` — the
  `.route("/metrics", get(routes::metrics::metrics_handler))` line is
  unconditional; when the feature is off the handler returns 404, so
  axum routing stays static. Avoids a `#[cfg]`-conditional `Router`
  build.
- `crates/nixfleet-control-plane/src/lib.rs` — gate `pub mod metrics;`.

The no-op stub pattern is preferred over `#[cfg]` at every call site
because `record_gate_block(...)` etc. are scattered across the CP and
gating each call would inflate the diff. Stubs keep call sites clean.

**LOC behind cfg:** 102 (metrics.rs) + 22 (routes/metrics.rs) = 124 LOC
optional when `metrics` is off. Default build is byte-identical.

### F4 — Merge `deferrals_view.rs` into `state_view.rs`

**Today:** Two top-level files for read-models served over HTTP:

- `state_view.rs` (257 LOC) — `fleet_state_view()` → `HostStatusEntry`
  rows for `/v1/hosts`.
- `deferrals_view.rs` (90 LOC) — `compute_channel_deferrals()` →
  `ChannelDeferral` rows for `/v1/deferrals` and `/metrics`.

Both:
- Read `state.verified_fleet`.
- Build `Observed` via `observed_view::build_for_gates_from_state`.
- Project domain state into wire-shape rows.

**After:** One file `state_view.rs` (~290 LOC), two public functions
exposed:

```rust
pub async fn fleet_state_view(state: &AppState) -> Result<Vec<HostStatusEntry>, StateViewError>;
pub async fn compute_channel_deferrals(state: &AppState) -> Vec<ChannelDeferral>;
```

**Mechanics:**

```bash
# Copy contents of deferrals_view.rs into state_view.rs (append),
# delete the standalone file, update mod declarations.
```

Call-site updates (2):
- `src/server/routes/deferrals.rs` — change
  `use crate::deferrals_view::...` to `use crate::state_view::...`.
- `src/metrics.rs` — same.

**LOC saved:** ~50 (module preamble + duplicate imports collapse).

## Test plan

**Per-item validation:**

- **F1** — extend `crates/nixfleet-cli/tests/cli_status_e2e.rs` with
  one new test exercising `nixfleet derive-pubkey` (deterministic
  output for a known fixture key). The existing `mint_token_smoke.rs`
  and `mint_operator_cert_smoke.rs` integration tests get renamed to
  exercise the new subcommand paths (`nixfleet mint-token`, etc.) —
  they already shell out, so updating one CLI invocation per test.

- **F2** — `cargo test -p nixfleet-control-plane --test lifecycle_parity`
  must pass after the move. The test logic is unchanged; only the
  imports differ.

- **F3** — three matrix runs:
  - `cargo build -p nixfleet-control-plane` (default features →
    metrics compiled in, identical binary).
  - `cargo build -p nixfleet-control-plane --no-default-features`
    (no metrics → smaller closure; `/metrics` returns 404).
  - `cargo test -p nixfleet-control-plane` (default features →
    full test suite passes).

- **F4** — `cargo test -p nixfleet-control-plane` must pass; the
  `/v1/deferrals` + `/v1/hosts` integration tests exercise both
  functions from the merged module.

**End-to-end gate (operator-run, not assistant-run):**

```
# Workspace test suite + clippy + host builds + rust-package builds.
nix run .#validate -- --rust
```

Must pass on the nixfleet PR before opening demo + fleet PRs. Heavy
commands (`cargo build --workspace`, `nix build`, full validate) are
deferred to the operator; the assistant produces the diff and stops
short of running them.

## Falsifiable done criteria

1. `${pkgs.nixfleet-cli}/bin/` contains exactly one binary (`nixfleet`)
   after the change. Verified via `ls $(nix build --no-link
   --print-out-paths .#nixfleet-cli)/bin/`.
2. `nixfleet mint-token --help`, `nixfleet mint-operator-cert --help`,
   and `nixfleet derive-pubkey --help` each produce the documented usage
   (no regression in operator UX).
3. `find crates/nixfleet-control-plane/src -name '*tests*.rs' -path '*/src/*'`
   returns nothing (no test files at module root in CP src tree).
4. `cargo build -p nixfleet-control-plane --no-default-features` does
   not pull `metrics` / `metrics-exporter-prometheus`. Verified via
   `cargo tree --no-default-features | grep -c metrics` == 0.
5. `grep -l 'mod deferrals_view' crates/nixfleet-control-plane/src/`
   returns nothing.
6. After demo + fleet PRs merge, `nix run .#regenerate-demo-identity`
   in the demo repo executes end-to-end without invoking any of the
   three dropped binary names.

## One-sentence summary

**Three operator binaries collapse into the `nixfleet` umbrella, one
test file moves out of `src/`, Prometheus becomes a default-on
feature flag, and two sibling HTTP read-models merge — ~1k LOC of
structural cleanup, no behavior change on default build.**
