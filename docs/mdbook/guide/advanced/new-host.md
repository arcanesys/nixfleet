# Adding a New Host

Step-by-step guide for adding a new machine to the config.

## 1. Add the Host Definition

In your fleet's `flake.nix`, add an `mkHost` call:

```nix
let
  mkHost = nixfleet.lib.mkHost;
  orgDefaults = { userName = "admin"; timeZone = "Europe/Paris"; };
in {
  nixosConfigurations.my-host = mkHost {
    hostName = "my-host";
    platform = "x86_64-linux";       # or "aarch64-linux", "aarch64-darwin"
    hostSpec = orgDefaults // {
      isDev = true;
      isImpermanent = true;
    };
    modules = [
      ./hardware/my-host/hardware-configuration.nix
      ./hardware/my-host/disk-config.nix
    ];
  };
}
```

For the framework test fleet, add entries in `modules/fleet.nix`.

## 2. Add Hardware Config

Create the hardware directory with:
- `hardware-configuration.nix` -- generated by `nixos-generate-config`
- `disk-config.nix` -- disko partitioning (use a template from `_shared/disk-templates/`)

## 3. Set Host Flags

Only declare what makes this host special. Smart defaults handle the rest:
- `isMinimal = true` suppresses base packages
- `isServer = true` marks as headless server
- `isImpermanent = true` enables ephemeral root

Fleet repos extend hostSpec with additional flags (e.g., isDev, isGraphical). Shared defaults are factored out via a `let` binding -- no need to repeat common values in each host.

## 4. Add Secrets

In the secrets repo:
1. Add the host's public key to `secrets.nix`
2. Re-encrypt secrets: `agenix -r`
3. Update: `nix flake update secrets`

## 5. Test

```sh
# Files must be tracked by git before evaluation
git add .

# Verify it evaluates
nix eval .#nixosConfigurations.my-host.config.system.build.toplevel.name

# Build it
nix build .#nixosConfigurations.my-host.config.system.build.toplevel

# Full validation
nix run .#validate
```

## 6. Deploy

```sh
# Fresh install (remote)
nixos-anywhere --flake .#my-host root@<ip>

# Rebuild (local or remote)
sudo nixos-rebuild switch --flake .#my-host

# macOS
darwin-rebuild switch --flake .#my-host
```
