# NixFleet

Declarative NixOS fleet management. Three layers, one wire protocol, no daemons on the agent path.

## What this book is

Two halves, two tools:

- **Curated guides** (this book) - architecture, the operator cookbook, troubleshooting, RFCs. Hand-written Markdown rendered by **mdbook**. Touched as rarely as the design changes.
- **Rust API reference** - generated by **`cargo doc --document-private-items`**. Type signatures, per-field and per-variant docs, resolved cross-references, source links, search. Lives at `./api/`. See [Rust API](api.md) for entry points.
- **NixOS option reference** - generated by **`pkgs.nixosOptionsDoc`** from the option declarations under `modules/`. See [Module options](options.md).

If you're new to the codebase: read [architecture](architecture.md) first, then [RFC-0003](rfcs/0003-protocol.md) for the wire protocol, then dip into the Rust API for whichever module you're touching.

## Building locally

```sh
# Regenerate everything: cargo doc + mdbook build. Idempotent.
nix run .#docs

# Outputs:
#   docs/mdbook/book/         - mdbook-rendered guides + RFCs
#   docs/mdbook/book/api/     - cargo doc Rust API reference
```

## Editing

Files in `docs/mdbook/src/` are plain Markdown. Edit them, push, the next `nix run .#docs` rebuilds the book. The Rust API regenerates from `///` comments - edit the Rust source, rebuild, your changes show up.
